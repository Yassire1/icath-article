Our empirical results reveal patterns of TSFM performance variation on industrial data. We categorize the observations into the following dimensions:

\subsection{Distribution Shift}

TSFMs are typically pretrained on diverse time series datasets including weather, traffic, and economic indicators. These domains exhibit fundamentally different statistical properties than industrial sensor data:

\begin{itemize}
    \item \textbf{Seasonality patterns}: Industrial data often contains multi-scale operational cycles that differ from natural seasonal patterns.
    \item \textbf{Noise characteristics}: Sensor noise is frequently non-Gaussian and correlated across channels.
    \item \textbf{Degradation dynamics}: Equipment degradation follows physics-driven monotonic trends absent from pretraining corpora.
\end{itemize}

\subsection{Deployment Efficiency}

We compare model size, GPU memory footprint, and inference latency for each evaluated model. Based on these measurements, we propose a practical deployment decision guideline for edge versus cloud PdM scenarios:

\begin{enumerate}
    \item Evaluate zero-shot performance on domain-specific held-out data before deployment
    \item Apply chronological train/test splitting to avoid data leakage
    \item Consider few-shot LoRA adaptation with limited labeled data when zero-shot performance is insufficient
    \item Select model size based on available edge compute constraints
\end{enumerate}
